ExchangeList:
	1. Nirvana.EsperRequest - Durable, fanout("//will be converted to direct")
			Used by prana.expnlCalculator for sending messages like "initialization inforamtion", "live feed request"
			listeners are:
				1. Prana.ExposureAndPnLCalulator for sending data
				2. Prana.OptionServer for recieving livefeed request
	
	2. Nirvana.Portfolio - Durable, Direct, routing key will be compressionLevel
			Used by prana.expnlCalculator for sending portfolio (post trade)
			Listeners are:
				1. prana.riskManagementJava for checking post trade rules
				2. PM for displaying data
	
	3. Nirvana.WhatIfPortfolio - Durable, Direct, routing key will be compressionLevel
			Used by prana.expnlCalculator for sending portfolio (pre trade)
			Listeners are:
				1. prana.riskManagementJava for checking post trade rules
				
	4. Nirvana.OverrideRuleRequest - Durable,Direct, routing key will be userId of client
			used by Prana.Server for asking client (whether to pass what if trade which has been blocked by rules)
			Listeners are:
				1. Prana.Client for responding 
				
	5. Nirvana.RuleBuildRequest - Durable,fanout
			Used by Prana.Client for risk management UI to refresh and build rule definition
			Listeners are:
				1. prana.riskManagementJava to build package for refreshing new rules
				
	6. Nirvana.RuleBuildResponse - Durable, fanout
			Used by prana.riskManagementJava to respond back for all clients tha rule definitions has be refreshed
			Listeners are:
				1. Prana.Client for refreshing their rule definition UI
	
	7. Nirvana.RuleSixteenB - Durable, fanout
			used by Prana.ExposureAndPnLCalulator for sending data at startup.
			used by Prana.Client for sending data at runtime.
			Listeners are:
				1. Prana.ExposureAndPnLCalulator for recieving rules at runtime
				2. prana.expnlCalculator for recieving rules at runtime for calculating runtime rulfeed genaration
